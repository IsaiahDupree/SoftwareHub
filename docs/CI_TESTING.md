# CI Testing Configuration

## Overview

SoftwareHub uses GitHub Actions for continuous integration testing. Tests run automatically on every push to `main` or `develop` branches, and on all pull requests.

## Workflow Jobs

### 1. Lint (`lint`)
- **Purpose**: Enforces code style and quality standards
- **Command**: `npm run lint`
- **Runs on**: Ubuntu Latest
- **Fast fail**: Yes

### 2. Type Check (`typecheck`)
- **Purpose**: Validates TypeScript types
- **Command**: `npx tsc --noEmit`
- **Runs on**: Ubuntu Latest
- **Fast fail**: Yes

### 3. Unit Tests (`test`)
- **Purpose**: Runs all Jest unit and integration tests
- **Command**: `npm run test -- --coverage --shard=$SHARD/3`
- **Runs on**: Ubuntu Latest
- **Parallelization**: **3 shards** running concurrently
- **Environment Variables**:
  - `NEXT_PUBLIC_SUPABASE_URL`: Local test database URL
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY`: Test anon key
  - `SUPABASE_SERVICE_ROLE_KEY`: Test service key
  - `NEXT_PUBLIC_META_PIXEL_ID`: Test pixel ID
  - `META_CAPI_ACCESS_TOKEN`: Test CAPI token
  - `RESEND_API_KEY`: Test email API key

### 4. Test Summary (`test-summary`)
- **Purpose**: Aggregates test results from all shards
- **Depends on**: `test`
- **Always runs**: Even if tests fail
- **Output**: GitHub Step Summary with shard status table

### 5. Build (`build`)
- **Purpose**: Validates that the application builds successfully
- **Command**: `npm run build`
- **Depends on**: `lint`, `typecheck`, `test-summary`
- **Artifacts**: Uploads `.next` build output (1 day retention)

### 6. E2E Tests (`e2e`)
- **Purpose**: Runs Playwright end-to-end tests
- **Command**: `npm run test:e2e -- --project=chromium`
- **Depends on**: `build`
- **Browsers**: Chromium only (for speed)
- **Artifacts**: Playwright HTML report (7 day retention)

## Parallel Testing

### Test Sharding
Tests are split across **3 parallel shards** to reduce CI time:

```bash
# Shard 1/3
npm run test -- --shard=1/3

# Shard 2/3
npm run test -- --shard=2/3

# Shard 3/3
npm run test -- --shard=3/3
```

Jest automatically distributes test files evenly across shards based on file count.

### Benefits
- **~66% faster test execution** compared to sequential
- Early failure detection across different test suites
- Better resource utilization

## Test Reports

### Coverage Reports
- **Generated by**: Jest with `--coverage` flag
- **Format**: LCOV, HTML
- **Location**: `coverage/` directory
- **Uploaded to**: Codecov (with shard-specific flags)
- **Retention**: 7 days in GitHub Actions artifacts

### Test Results
- **Format**: JUnit XML (optional), Jest JSON
- **Location**: `test-reports/` directory
- **Artifacts**: `test-results-shard-{1,2,3}`
- **Retention**: 7 days

### Playwright Reports
- **Format**: HTML with screenshots and traces
- **Location**: `playwright-report/` directory
- **Artifacts**: `playwright-report`
- **Retention**: 7 days

### GitHub Step Summary
Each CI run generates a markdown summary showing:
- Test shard status (✅ passed / ❌ failed)
- Build status
- Coverage links (when integrated with Codecov)

## Environment Variables

### Required Secrets (GitHub)
Configure these in **Repository Settings → Secrets**:

| Secret Name | Purpose | Example |
|-------------|---------|---------|
| `NEXT_PUBLIC_SUPABASE_URL` | Production Supabase URL | `https://xxx.supabase.co` |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Production anon key | `eyJhbGc...` |

### Test Environment Variables
These are hardcoded in the workflow for local test environment:

```yaml
env:
  NEXT_PUBLIC_SUPABASE_URL: http://127.0.0.1:54821
  NEXT_PUBLIC_SUPABASE_ANON_KEY: eyJhbGc...  # Local demo key
  SUPABASE_SERVICE_ROLE_KEY: eyJhbGc...      # Local demo key
```

## Running Tests Locally

### All tests (sequential)
```bash
npm run test
```

### With coverage
```bash
npm run test:coverage
```

### Specific shard (parallel)
```bash
npm run test -- --shard=1/3
npm run test -- --shard=2/3
npm run test -- --shard=3/3
```

### E2E tests
```bash
npm run test:e2e
```

## Troubleshooting

### Tests fail in CI but pass locally
1. Check environment variables are set correctly
2. Verify Node.js version matches (v20)
3. Clear `node_modules` and reinstall: `npm ci`
4. Check for timezone/locale-specific tests

### Shard failures
- Each shard runs independently
- Check individual shard artifacts for failure details
- Shards may fail if tests have interdependencies (anti-pattern)

### E2E test failures
- Download Playwright report artifact from failed run
- Open `index.html` to see screenshots and traces
- Check for timing issues (increase timeouts if needed)

## Performance Tips

### Reducing Test Time
1. **Use test sharding** (already enabled with 3 shards)
2. **Skip E2E on draft PRs**: Add `if: github.event.pull_request.draft == false`
3. **Cache dependencies**: Already using `cache: 'npm'`
4. **Optimize slow tests**: Use `jest --onlyChanged` for incremental runs

### Cost Optimization
- E2E tests only run on `chromium` (not all browsers)
- Artifact retention limited to 7 days
- Tests use `maxWorkers=2` to avoid OOM errors

## Maintenance

### Updating Node.js Version
Update `NODE_VERSION` environment variable at the top of `.github/workflows/ci.yml`:

```yaml
env:
  NODE_VERSION: '20'  # Change to new LTS version
```

### Adding New Test Jobs
Follow this pattern:

```yaml
new-test-job:
  name: New Test Job
  runs-on: ubuntu-latest
  needs: [lint, typecheck]  # Dependencies
  steps:
    - uses: actions/checkout@v4
    - name: Setup Node.js
      uses: actions/setup-node@v4
      with:
        node-version: ${{ env.NODE_VERSION }}
        cache: 'npm'
    - run: npm ci
    - run: npm run test:new-type
```
